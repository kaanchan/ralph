# 🤖 Project Protocols

## Why is this directory at the root?
This directory is deliberately placed at the root level (rather than nested under `docs/`) because it serves a specialized purpose: **Instructions for Automation Tools & AI Agents**.

While the `docs/` directory is primarily intended for human consumption (architectural overviews, developer Q&A, setup guides), the `protocols/` directory contains strict operational standards that AI agents (like RALPH or Antigravity) must adhere to when modifying the repository. 

By placing it at the root, it signals to any future agent parsing the repository tree that these files contain non-negotiable operational rules, rather than passive human-readable documentation.
