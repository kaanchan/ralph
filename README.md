# RALPH (Recursive Agentic Local Python Hacker)

RALPH is a local-first recursive coding agent. Given a task pod, it iteratively plans, executes, evaluates, and loops until the test suite passes. It uses free local 14B models (like `qwen25-coder-14b`) via direct Ollama API for code generation, and falls back to Aider + Gemini when local attempts fail.

## Architecture: Task Pods

Each task lives in an isolated **Task Pod** under `tasks/`:

```
tasks/
  001_fibonacci/
    task.json          # goal description and metadata
    topology.py        # LangGraph graph definition for this task
    graph.json         # graph config
    src/
      solution.py      # generated by RALPH (reset to placeholder each run)
      test_solution.py # the SPEC — never modified by RALPH
    checkpoints.sqlite # LangGraph run state (cleared on reset)
    traces/            # per-run execution traces (cleared on reset)
```

`src/solution.py` and `src/test_solution.py` are the only files the agent reads and writes. The test spec is authored once and never touched by RALPH. The solution starts as `# generated by ralph` and is overwritten each iteration.

## 📁 Project Structure

- **`src/`**: Core application logic.
  - `main.py`: CLI entry point — `run <task_dir>` or `history`.
  - `graph.py`: Base LangGraph loop skeleton.
  - `nodes/`: Individual node implementations (planner, executor, evaluator, router, escalator).
  - `hardware.py`: Dynamic VRAM profiler via `nvidia-smi`.
  - `model_registry.py`: Model capability and VRAM baseline registry.
  - `config.py`: Centralized config; `set_task_directory()` re-routes paths per pod.
  - `runlog.py`, `telemetry.py`, `state_exporter.py`: Logging and observability.
- **`tasks/`**: Task pods (see above).
- **`scripts/`**: Utility scripts — `reset-task.ps1` resets a pod to clean state.
- **`tests/`**: Unit tests for individual components.
- **`diagnostics/`**: Layered smoke test suite for environment validation before a heavy run.
- **`docs/`**: Architecture research and handoff documents.
- **`prompts/`**: System prompt templates.
- **`plans/`**: Versioned implementation plans from agentic planning sessions.

## 🚀 Running RALPH

Ensure `.env` is populated and `.venv` is active.

```powershell
# Run a task pod
python src/main.py run tasks/001_fibonacci/

# View run history
python src/main.py history
```

### Reset a task pod before re-running

```powershell
pwsh -File scripts\reset-task.ps1 001_fibonacci
```

This resets `src/solution.py` to the placeholder, deletes `checkpoints.sqlite`, `traces/`, and any stale aider logs. The test spec is never touched.
